# Blackfire Crusade 自動化副本掛機輔助工具 🎮

這是一個基於 **電腦視覺（Computer Vision / OpenCV 模板匹配）** 與 **GUI 自動化（PyAutoGUI）** 技術開發的 2D 回合制遊戲《Blackfire Crusade》自動副本掛機腳本。

本工具專為 PC 視窗模式（如 Steam 版本）設計，能自動偵測遊戲狀態（準備大廳、戰鬥中、結算畫面）並執行對應的操作（點擊開始、開啟自動戰鬥、點擊繼續與再次挑戰），省去手動重複點擊的時間。

---

## ✨ 核心特色與技術

1. **極速與輕量化**：
   * 採用 `mss` 高效畫面擷取，單次截圖僅需 10~30ms，CPU 佔用率極低（$< 1\%$）。
   * 支援 `PIL ImageGrab` 作為備份截圖方案，確保系統相容性。
2. **閉環反饋與 CD 重試控制 (Auto Toggle)**：
   * 透過比對未啟用的「自動戰鬥」模板，並配合 3 秒冷卻時間，可確保遊戲在延遲或漏點時自動重試，同時絕不會發生「重複點擊而將已啟用的自動戰鬥關閉」的問題。
3. **動態多段結算處理 (Multi-Continue Winner)**：
   * 自動搜尋並快取 `templates/common/continue*.png` 等所有繼續按鈕。
   * 比對時採取「相似度 PK 機制」，僅會點選畫面中相似度最高的有效前台按鈕，完全無視背景殘留變暗按鈕的干擾，順暢通過多重結算頁面。
 4. **自動定時領體力 (Stamina / Bread)**：
    * 支援啟動時與每隔 30 分鐘自動進行體力領取（透過 `door` ➔ `bread` ➔ `bread_collection` ➔ `confirm` ➔ `quit_bread` 系列按鈕）。
    * 具備體力已滿提示自動跳過容錯處理，絕不卡死。
4. **自動定時領體力 (Stamina / Bread)**：
   * 支援啟動時與每隔 30 分鐘自動進行體力領取（透過 `door` ➔ `bread` ➔ `bread_collection` ➔ `confirm` ➔ `quit_bread` 系列按鈕）。
   * 具備體力已滿提示自動跳過容錯處理，絕不卡死。
5. **自動定時領鑽石 (Diamond Claim)**：
   * 支援啟動時與每隔 2 小時自動進行免費鑽石領取。若偵測不到鑽石入口按鈕，會自動點選 `goback_town` 返回大廳，再依次點擊 `diamond` ➔ `diamond_free` ➔ `confirm` ➔ `quit_bread` 系列按鈕。
   * **領取優先順序**：當體力與鑽石定時器同時到期時，腳本將**優先領取鑽石，隨後才領取體力**，最終返回掛機，以確保最高的領取成功率。
6. **地下城探索記憶與下樓過渡冷卻 (Action Memory & Godown Cooldown)**：
   * 地圖事件（寶箱、技能事件、接受祝福）一旦被點選，程式**立即標記該事件本層已完成**，並在後續截圖中完全跳過，解決圖示點完不消失導致的重複點擊問題。
   * 下樓按鈕被點選後，自動開啟 **6 秒過渡期冷卻計時**，冷卻未完成前絕不重置探索記憶，防止載入下一層的時間差內重複點選舊圖示。
7. **滑鼠手動介入偵測與鎖定狀態暫停 (Anti-Interference Manual Pause)**：
   * 智慧檢測玩家的滑鼠操作。若滑鼠發生偏移（$> 5$ 像素）且距離腳本上一次操作超過 1.2 秒，程式會**自動暫停掛機**並鎖定目前狀態。
   * **特別說明**：本偵測**僅針對滑鼠移動**進行攔截；玩家使用鍵盤操作（如打字、熱鍵）時，腳本仍會照常自動偵測與執行，不會因此暫停。
   * 當玩家雙手離開滑鼠、滑鼠靜止達 3 秒後，腳本將自動恢復掛機並繼續原動作。
8. **全域「任務完成」獎勵自動領取 (Global Task Reward Claim)**：
   * 回到大廳時，若彈出任務完成彈窗 (`templates/task_complete.png`)，程式會進行全域攔截。
   * 利用精準計算的相對比例位移，自動點選彈窗下方的「領取獎勵」按鈕（X 軸中心，Y 軸向下偏移 281 像素），並在大廳狀態下自動點擊確認，無人值守更省心。
9. **模組化狀態機設計**：
   * 採用**狀態模式 (State Pattern)** 重構，狀態處理邏輯完全與主調度器解耦，易於擴充新狀態。
10. **防作弊模擬點擊**：
    * 滑鼠移動模擬人類平滑曲線軌跡，並加入微幅的隨機像素偏移與點擊冷卻，保護遊戲帳號。

## 🛠️ 環境配置與安裝

請確保您的電腦已安裝 **Python 3.7 或以上版本**，並在專案根目錄下使用終端機執行：

1. **建立虛擬環境**：
   ```powershell
   python -m venv .venv
   ```
2. **安裝依賴套件**：
   ```powershell
   .\.venv\Scripts\pip install -r requirements.txt
   ```

## 📖 詳細使用指南

請依照以下 3 個步驟進行設定與啟動：

### 步驟 1：🔍 確認您的遊戲視窗名稱
開啟遊戲後（不可最小化），執行：
```powershell
.\.venv\Scripts\python list_windows.py
```
確認您的遊戲視窗標題為 `Blackfire Crusade`（預設），若不同，請在步驟 3 中加入參數。

---

### 步驟 2：✂️ 裁剪您的遊戲專屬按鈕模板
執行以下命令啟動裁剪工具：
```powershell
.\.venv\Scripts\python crop_tool.py
```
1. 終端機將開始 **5 秒倒數**，請立即點擊切換到遊戲視窗，將畫面停留在含有目標按鈕的畫面上。
2. 5 秒後，程式會自動擷取遊戲視窗並彈出裁剪視窗。
3. **操作方式**：左鍵拖曳拉出一個矩形，選中按鈕後按下 `Enter` 或 `Space` 鍵確認選取。（按 `ESC` 鍵可取消並退出）
4. 在終端機中輸入對應的檔名保存，並將產生的圖片放入 `templates/` 下對應的子資料夾。

> 💡 **模板裁剪最佳實踐（避開背景變動）**：
> 裁剪時，請**儘可能縮小範圍，僅框選按鈕內固定的文字（例如只框「繼續」或「再戰」這兩個字）**，避免將按鈕周圍的發光特效、動態背景或陰影剪進去。這樣即使遊戲背景如何變化，程式都能 100% 辨識成功。

---

### 步驟 3：🚀 啟動自動掛機
執行以下指令開始掛機：

* **普通關卡模式**：
  ```powershell
  .\.venv\Scripts\python main.py --mode stage
  ```
  > ⚠️ **注意事項**：啟動前請**先手動選好關卡**，並將遊戲畫面停留在**進入戰鬥之前的畫面（即能看見"去戰鬥" `stages/start.png` 按鈕的畫面）**，腳本才能正常開始掛機。

* **史萊姆地下城模式**：
  ```powershell
  .\.venv\Scripts\python main.py --mode dungeon_slime
  ```
  > 💡 **注意事項**：地下城模式支援自動尋路導航。啟動前只需確保遊戲畫面在**最外層的主畫面（能看到 `common/door.png` 入口按鈕的畫面）**即可，腳本會自動點擊入口並一路尋路進入副本。


* **可用自訂參數**：
  * `--title "視窗標題"`：如果您的遊戲視窗名稱不叫 `Blackfire Crusade`。
  * `--interval 秒數`：調整畫面偵測頻率。預設為 `0.5` 秒。
  * *範例*：`.\.venv\Scripts\python main.py --mode dungeon_slime --interval 0.5`

* **終止程式**：在終端機隨時按下 `Ctrl + C`，程式會安全退出，並為您統計本次掛機一共完成的場次。

---

### 🛡️ 手動操作介入與暫停機制說明

當您在自動掛機過程中需要臨時手動操作遊戲時，**不需關閉腳本**，只需直接移動滑鼠即可：

1. **觸發暫停**：
   * 只要**手動移動滑鼠**（位移大於 5 像素），程式會判定為手動操作，**立即暫停自動決策與模擬點擊**，並鎖定與保留目前的遊戲狀態（例如保持在戰鬥中或探索中）。
   * **重要提示**：此暫停機制**僅針對滑鼠移動進行檢測**。如果您使用鍵盤操作（如打字聊天、按熱鍵），程式**不會觸發暫停**，仍然會照常偵測畫面並執行自動掛機。
2. **自動恢復**：
   * 當您手動操作完畢，**鬆開滑鼠並保持靜止 3 秒後**，程式將會自動感應並列印日誌，無縫恢復自動掛機決策，繼續原本的工作。
3. **安全防護**：
   * 系統已內建防自點判定。當腳本點擊滑鼠時，會自動在接下來的 1.2 秒內忽略檢測，絕不會把「腳本本身的點擊移動」誤判為玩家手動介入。
